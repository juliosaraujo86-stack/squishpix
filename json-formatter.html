Dev guide

Common JSON errors & fixes

Single quotes

{'key': 'val'}

JSON requires double quotes for all strings and keys.

Trailing comma

{"a":1,"b":2,}

A comma after the last item causes a parse error. Remove it.

Unquoted keys

{key: "value"}

All JSON keys must be in double quotes. Valid JS, not valid JSON.

Comments

{"a":1 // note}

JSON doesn't support comments. Remove all // or /* */.

Unclosed brackets

{"a":[1,2,3}

Every opening bracket must have a matching closing one.

Undefined / NaN

{"v":undefined}

undefined, NaN, Infinity are not valid JSON. Use null.
